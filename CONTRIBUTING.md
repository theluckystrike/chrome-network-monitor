# Contributing to chrome-network-monitor

Thanks for your interest in contributing. This document covers the basics for getting involved.

GETTING STARTED

1. Fork the repository on GitHub
2. Clone your fork locally
3. Install dependencies with npm install
4. Create a feature branch from main

```bash
git checkout -b feature/your-feature-name
```

DEVELOPMENT WORKFLOW

Build the project with TypeScript.

```bash
npm run build
```

Make sure the build passes before submitting a pull request.

SUBMITTING CHANGES

1. Keep commits focused and atomic
2. Write clear commit messages that explain what changed and why
3. Push your branch to your fork
4. Open a pull request against main
5. Describe what your PR does and link any related issues

REPORTING BUGS

Use the GitHub issue tracker. Include steps to reproduce, expected behavior, and actual behavior. Browser version and extension manifest version are helpful for debugging.

CODE STYLE

- TypeScript strict mode is enabled
- Follow the existing patterns in the codebase
- Keep functions small and well-named
- Add JSDoc comments to public methods

QUESTIONS

Open a discussion or issue on GitHub if something is unclear.
